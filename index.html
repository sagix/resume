Intégration continue et qualité

Au début du projet, presque qu'aucun outil n'était en place pour construire les livrables, et pour le peu qu'il y avait, personne ne lui faisait confiance.
Un objectif de cette mission était de pallier à cette situation en mettant en place un système de build d'intégration continue, ainsi que des outils de mesure de qualité.
Pour le build, ant a été remplacé par Gradle et au niveau IDE, Eclipse par Android Studio. Gradle étant plus souple, il a été plus facile de construire des applications pour les différents stores, à l'aide des flavours.
Puis pour l'intégration, Jenkins a été mise à niveau et publie automatiquement les livrables sur HockeyApp à chaque push sur git. Afin qu'une version dogfood reflète le développement courant, et que les releases puissent être testables facilement.
À ce moment, des mesures sont faites avec lint, Checkstyle, PMD et FindBugs.
De plus, les tests unitaires sont exécutés par JUnit, et la couverture de code est mesurée par JaCoCo.
Tous ces outils permettent de garder une qualité constante. La suite sera d'automatiser les tests fonctionnels.

Bouygues Telecom Espace Client Mobile

Site web mobile disponible pour toutes les lignes de marché.

Il évolue pour rattraper son retard sur les technologies web, les tendances de design et les "nouveaux" formats d'écran.

Un important travail a été fait sur la partie présentation, avec la création d'un build grunt, servant à compiler et fusionner un framework CSS se basant sur la méthodologie SMACSS et le pré-processeur Less. Ainsi que l'assemblage d'un framework javascript fusionnant des fichiers javascript en les optimisant et passant des tests unitaires. Une validation syntaxique est passée tant sur le CSS que sur le javascript.

Un second travail a été fait sur les images pour qu'elles soient responsives et s'adaptent à la densité de l'écran.

Ce travail et cette recherche a donné lieu à une présentation sur Responsive Web Design.

Derrière tout ça le site est fait en jee/java7, avec JSF et tournant sur jboss7, avec un build comprenant de nombreux tests unitaires.
